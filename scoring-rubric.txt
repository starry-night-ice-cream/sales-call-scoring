ANALYSIS REQUIREMENTS:

1. Identify Primary Use Case.
2. Identify Secondary Use Case(s).
3. Identify Required Data Products.
4. Identify Current State (competitor, build, status quo).
5. Evaluate BANT clearly (Present / Partial / Missing).
6. Score each Discovery category (1–10).
7. Score overall Objection Handling (1–10).
8. Identify:
   - Fatal Flaws
   - Illusion Signals
9. Recommend:
   - Stage Movement (Forward / Stay / Backward / Disqualify)
   - Amount Adjustment (Increase / Maintain / Decrease / Unknown)
10. Detect expansion potential across additional use cases.
11. Generate Stage-Adjusted Close Probability (0–100%).
    - Early stages should not exceed 40% without strong BANT.
    - Late stages require confirmed authority + timeline to exceed 70%.

OUTPUT FORMAT:

Render the entire audit as ONE visualizer widget (visualize:show_widget,
HTML mode, dark theme, transparent background). Do NOT produce the audit
as prose or markdown — the widget IS the deliverable.

Required widget sections, in order:

1. HEADER
   - Eyebrow: "Post-call audit"
   - Company name (large)
   - Tag pill (top-right): one of "New Business", "Re-engagement / Expansion",
     "Renewal", "Competitive Replace", "Disqualify Candidate"
   - Sub-line: "{Meeting Date} · {Vendor attendee} → {Prospect attendee}"

2. EXECUTIVE RECAP
   - Bordered card, ≤150 words narrative, left accent stripe.

3. KPI ROW (4 cards, equal width)
   - Current amount (white/neutral)
   - Recommended amount (green if increase, red if decrease, neutral if maintain)
   - Close probability (color-coded: <35% yellow/orange, 35–65% blue, >65% green)
   - Stage action (color-coded: Forward green, Stay neutral, Re-stage/Backward
     orange, Disqualify red)

4. BANT + DISCOVERY SCORES (two cards side-by-side)
   - BANT: Budget / Authority / Need / Timeline, each with a colored pill
     (Missing = red, Partial = yellow, Present = green).
   - Discovery: 6 horizontal bars (Latent pain, Active pain, Quantified impact,
     Solution dev, Evaluation, Decision), each 0–10 with score label on right.
     Bar color graduated: 0–3 red, 4–6 yellow, 7–10 green.

5. FATAL FLAWS + ILLUSION SIGNALS (two cards side-by-side)
   - Fatal flaws: red/pink heading, bulleted list.
   - Illusion signals: orange/amber heading, bulleted list.

6. DEAL INTELLIGENCE (single card, two-column grid)
   - Primary use case | Current state
   - Secondary use cases | Objection handling score (X/10)
   - Required products | Expansion potential

7. TOP 3 NEXT QUESTIONS (card, numbered list)

8. TOP 3 EXECUTIVE MOVES (card, numbered list, bold lead-in + supporting detail)

STYLING NOTES FOR THE WIDGET:
- Dark theme. Use CSS variables; transparent outer background.
- Cards: subtle border, generous padding, rounded corners.
- Pills: rounded-full, padded, semibold caps for status labels.
- Typography: tight headings, comfortable body line-height.
- No emojis, no marketing tone. Operator voice only.
